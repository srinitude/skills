# Support

Use a GitHub issue for reproducible installation, validation, packaging, or client-integration problems.

Include:

- the repository version or commit;
- the client and its version;
- the exact command;
- the full error output with secrets removed;
- whether `mise run ci` passes in the checkout.

Use the security process in [SECURITY.md](SECURITY.md) for private reports.

Questions about the skill procedure belong in an issue. Questions about a client product belong in that product's support channel when this repository's manifest is not involved.
